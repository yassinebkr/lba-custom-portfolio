Sprite color diagnostic — direct load, no React, no caching

Each row loads a fresh copy of the PNG (cache-busted) and measures its actual pixels with a canvas.

Verdict rule: "dark" = avg < 100, "mid" = 100-180, "white" = > 180. Anything not marked "white" is not actually white.

If a sprite shows "dark" / "mid" here but still looks white in the main UI, the main UI is serving a stale cached file. Hard-reload (Ctrl+Shift+R) the main UI.
